# Пошаговая инструкция деплоя на Railway

## Шаг 1: Коммит и пуш в Git

```bash
cd hyperion-site
git add .
git commit -m "Настройка для Railway"
git push
```

## Шаг 2: Подключение репозитория в Railway

1. Зайдите на [railway.app](https://railway.app)
2. Нажмите **"New Project"**
3. Выберите **"Deploy from GitHub repo"**
4. Выберите ваш репозиторий
5. Railway начнет автоматический деплой

## Шаг 3: Добавление переменных окружения

**ВАЖНО:** Добавьте переменные **ДО** того, как первый деплой завершится, или сразу после подключения репозитория.

1. В Railway откройте ваш проект
2. Перейдите в **"Variables"** (вкладка вверху)
3. Добавьте следующие переменные:

```
ADMIN_USERNAME = admin
ADMIN_PASSWORD = ваш_надежный_пароль
SESSION_SECRET = случайная_длинная_строка_минимум_32_символа
NODE_ENV = production
```

**Как сгенерировать SESSION_SECRET:**
- Можно использовать: `openssl rand -hex 32`
- Или любой генератор случайных строк (минимум 32 символа)

## Шаг 4: Ожидание деплоя

Railway автоматически:
- ✅ Установит зависимости (`npm install`)
- ✅ Соберет проект (`npm run build`)
- ✅ Запустит сервер (`npm start`)

## Шаг 5: Проверка

1. Railway предоставит URL вашего приложения (например: `your-app.up.railway.app`)
2. Откройте URL в браузере
3. Перейдите на `/login`
4. Войдите с учетными данными из переменных окружения

## Порядок действий (кратко):

```
1. git add . && git commit -m "..." && git push
   ↓
2. Railway → New Project → Deploy from GitHub
   ↓
3. Railway → Variables → Добавить переменные
   ↓
4. Ждать завершения деплоя
   ↓
5. Открыть URL и проверить работу
```

## Если что-то пошло не так:

- **Проверьте логи** в Railway (вкладка "Deployments" → выберите деплой → "View Logs")
- **Убедитесь**, что все переменные окружения добавлены
- **Проверьте**, что код успешно запушен в Git
